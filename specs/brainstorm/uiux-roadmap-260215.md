# Auto-Sub UI/UX Improvement Roadmap

> 腦力激盪綜合報告 — 2026-02-15
>
> 由 Proposer（方案提出）、Researcher（業界研究）、Devil's Advocate（批判審視）三方意見整合

---

## 執行摘要

Auto-Sub 的核心價值是**即時翻譯字幕**。所有 UI/UX 改進都應直接服務這個核心體驗。經過三方討論，我們將原始 7 個提案精煉為 **5 個務實改進項目**，按 MVP 原則大幅縮減範圍，優先解決真正影響使用體驗的問題。

---

## Tier 1：建議執行（高 ROI、低風險）

### 1.1 字幕可讀性微調（原 P0-1 縮減版）

**問題**：現有字幕在部分場景下可讀性不足。

**批判後調整**：
- ~~新增字幕樣式預設~~ → 不做，避免設定膨脹
- ~~自訂文字顏色~~ → 不做，目前色彩方案已夠用
- ~~NSAttributedString 替代 8 層描邊~~ → 不做，現有方案穩定可預測，換架構風險大
- ~~背景模糊效果~~ → 不做，效能開銷大且需重構視窗結構

**保留的改進**：
1. **驗證並微調現有對比度**：確認目前 `Color.black.opacity(0.7)` + 白字描邊在各種場景下的實際對比度，必要時調高背景不透明度預設值
2. **背景透明度 Slider 的預設值優化**：目前 `subtitleWindowOpacity` 預設 0.7，可根據測試調整到更可讀的值
3. **字幕最大寬度限制**：避免過長文字橫跨全螢幕（Researcher 建議，低成本高效益）

**相關檔案**：`SubtitleOverlay.swift:107-110`、`SubtitleWindowController.swift:120-121`
**難度**：低
**依據**：WCAG 對比度概念（雖非強制，但作為可讀性參考）、Researcher 字幕可讀性研究

---

### 1.2 停止擷取後保留字幕（原 P0-2 縮減版）

**問題**：停止擷取時字幕瞬間消失，使用者無法回顧最後幾句。

**批判後調整**：
- ~~「暫停模式」~~ → 不做，會讓狀態機過於複雜，且與「保留 N 秒」語意重疊
- ~~可配置保留時間~~ → 不做，先用固定值，避免增加設定項

**保留的改進**：
1. **停止後保留字幕 3 秒 + 淡出**：`stopCapture()` 改為先觸發 fade-out 動畫（2-3 秒），動畫完成後再清理
2. **處理邊界條件**：若淡出過程中使用者重新開始擷取，立即取消動畫並恢復正常模式

**相關檔案**：`MenuBarController.swift:556-575`
**難度**：低
**注意**：Devil's Advocate 指出「服務已停但 UI 還在」的不一致——解法是 UI fade-out 與服務停止同時啟動，只是視覺上延遲隱藏，不影響服務狀態

---

### 1.3 Menu Bar 擷取時長顯示（原 P1-2 縮減版）

**問題**：擷取中時，使用者從 Menu Bar 無法得知系統運行狀態。

**批判後調整**：
- ~~脈動動畫~~ → 不做，違反 Apple HIG（Menu Bar icon 應靜態）
- ~~翻譯句數、統計區塊~~ → 不做，使用者不需要這些開發者指標
- ~~Gemini token 使用量~~ → 不做，技術指標非使用者關注點

**保留的改進**：
1. **Status row 顯示擷取時長**：在 `StatusMenuItemView` 的擷取中狀態旁顯示已擷取時長（如「擷取中 · 05:32」）
2. **實作方式**：在 `AppState` 記錄 `captureStartTime`，menu 打開時計算差值顯示，避免 Timer 持續更新

**相關檔案**：`MenuBarController.swift:326-335`、`MenuBarController.swift:578-633`
**難度**：低
**依據**：Apple HIG - Menu Bar icon 應反映應用狀態

### 1.4 字幕視窗直接拖拉調整大小（原 P1-3 改進版）

**問題**：目前調整字幕框寬高需要進入 Settings → 字幕渲染 Tab → 拖移 Slider，操作路徑過長且不直覺。

**方案**：
- **取代** Settings 中的寬高 Slider，改為解鎖狀態下直接拖拉邊角調整大小
- 這是「減法設計」——移除間接控制，換成直接操作，操作路徑更短
- 只有一個控制途徑，不存在數值同步問題

**保留的改進**：
1. **解鎖時顯示虛線邊框 + 拖拉角標**：視覺回饋讓使用者清楚知道可調整大小
2. **拖拉邊角直接 resize**：自行實作 borderless window 的邊角拖拉手勢
3. **移除 Settings 中的寬高 Slider**：減少設定項，簡化設定介面

**不做的部分**：
- ~~長按觸發解鎖~~ → `ignoresMouseEvents = true` 下技術不可行
- ~~自動鎖定~~ → timing 魔數問題無法優雅解決
- ~~兩套控制並存~~ → 直接取代，避免同步問題

**相關檔案**：`SubtitleWindowController.swift:115`（borderless window）、`SubtitleOverlay.swift:22-36`（drag handle）、`SettingsView.swift`（待移除的 Slider）
**難度**：中
**依據**：macOS Live Captions 也支援直接拖角 resize；減法設計符合 MVP 原則

## Tier 3：推遲（低 ROI 或高風險）

| 原提案 | 推遲原因 |
|--------|---------|
| P0-3 Onboarding 改善 | 只用一次的流程，ROI 極低；權限檢測跨 macOS 版本不一致 |
| P1-1 設定介面即時預覽 | 需維護兩套渲染邏輯；移除固定視窗大小會破壞佈局穩定性 |
| P2-1 字幕匯出/歷史 | 全新功能模組，需要持久化架構變動；隱私顧慮；嚴重偏離 MVP |

## Researcher 建議的採納情況

| 建議 | 採納 | 原因 |
|------|------|------|
| 字幕半透明背景框 | ✅ 微調現有值 | 已有機制，只需調整預設 |
| Menu Bar icon 狀態反映 | ✅ 加擷取時長 | 符合 HIG，低成本 |
| Interim 文字視覺區分 | ⏸️ 觀察 | 目前已有顏色區分（橘色 vs 白色） |

| Fade-in 微動畫 | ✅ 融入 1.2 | 搭配停止淡出一起做 |
| 螢幕分享排除 | ❌ 推遲 | macOS 13 支援不完整 |
| 讀取系統字幕偏好 | ❌ 推遲 | 低優先級，Auto-Sub 有自己的設定 |
| 跨虛擬桌面顯示 | ✅ 已有 | `canJoinAllSpaces` 已設定 |

## 建議執行順序

```
Phase 1（1-2 天）
├── 1.2 停止後保留字幕 3 秒 + 淡出（最低成本、直接解決痛點）
└── 1.3 Menu Bar 擷取時長顯示（簡單加值）

Phase 2（1-2 天）
├── 1.1 字幕可讀性微調（調整預設值 + 最大寬度限制）
└── 1.4 字幕視窗直接拖拉 resize（取代 Slider，減法設計）

Phase 3（評估後決定）
└── 2.1 快捷鍵切換字幕鎖定（需先確認使用者需求）
```

## Devil's Advocate 的核心提醒

> 「這些提案大多是『工程師覺得酷』而不是『使用者真正需要』。Auto-Sub 的核心價值是即時翻譯字幕，任何不直接服務這個核心的改善都應該慎重評估 ROI。」

瑠瑠覺得這個提醒非常重要。在沒有使用者回饋數據之前，應該先把核心體驗做穩做好，而不是急著加功能。
